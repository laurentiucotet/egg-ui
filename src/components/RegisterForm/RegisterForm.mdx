# RegisterForm

A registration form component with full name, email, password, and confirm password fields, wrapped in a card component. The form includes built-in validation that prevents submission when required fields are empty.

## Usage

<Source language="tsx" code={`import { RegisterForm } from './RegisterForm'

// Basic usage
<RegisterForm onSubmit={(data) => console.log(data)} />

// With custom styling
<RegisterForm 
  onSubmit={(data) => console.log(data)} 
  className="w-full max-w-md mx-auto"
/>`} />

## RegisterFormData

The data shape returned when the form is successfully submitted.

| Property | Type | Description |
|----------|------|-------------|
| fullName | `string` | The user's full name |
| email | `string` | The user's email address |
| password | `string` | The user's password |
| confirmPassword | `string` | Password confirmation |

## Features

### Form Validation
- All fields (full name, email, password, and confirm password) are required
- Form submission is prevented if any field is empty
- Visual validation through HTML5 required attribute

## Best Practices

### Password Matching
While this component collects both password and confirmPassword fields, it's recommended to implement password matching validation in your application logic:

```tsx
<RegisterForm 
  onSubmit={(data) => {
    if (data.password !== data.confirmPassword) {
      // Show error message
      alert('Passwords do not match')
      return
    }
    // Proceed with registration
    console.log('Registration data:', data)
  }}
/>
```

### Password Strength
Consider adding password strength requirements in your application:
- Minimum length (e.g., 8 characters)
- Mix of uppercase and lowercase letters
- Include numbers and special characters

### Email Validation
While the input type="email" provides basic validation, consider adding additional email format validation in your submission handler.
